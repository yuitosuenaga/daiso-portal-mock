# Implementation Plan

- [x] 1. Foundation: デザイントークンをDAISOブランドカラーに更新する
  - プライマリカラー・アクセントカラー・本文文字色・フォーカスリング色の変数値をDAISOピンク基調の値へ更新する
  - エラー系・成功系の色は現在の色相を維持し、ブランドカラーと混同されないことを確認する
  - 変更後、既存のセマンティッククラス（塗り・淡色背景・文字色）を使う画面がビルドエラーなく新しい配色で表示される状態になる
  - _Requirements: 1.1, 1.2, 1.4, 4.1_

- [x] 2. Core: 共通UIコンポーネントにDAISOブランドのバリアントを追加する
- [x] 2.1 (P) ボタンのアウトライン表現をブランドカラー基調に変更する
  - 枠線・文字色をブランドカラーにし、ホバー時に塗り＋白文字へ反転する見た目にする
  - 既存の塗りボタン（primary）はデザイントークン変更を通じて自動的にブランドカラーになることを確認する
  - ボタンにマウスホバーした際、枠線表示から塗り表示へ切り替わる状態を目視で確認できる
  - _Requirements: 1.3, 2.1, 2.2_
  - _Boundary: Button_

- [x] 2.2 (P) バッジに汎用ブランド表現用のバリアントを追加する
  - ブランドカラー塗り・白文字の新しいバリアントを既存バリアントと衝突しない形で追加する
  - 既存の用途別バリアント（メンテナンス通知・ポリシー・障害・その他・状況・緊急度）の見た目・呼び出し方法を変更しない
  - 新バリアントを指定したバッジがブランドカラー塗りで表示される状態を確認できる
  - _Requirements: 1.3, 2.3_
  - _Boundary: Badge_

- [x] 2.3 (P) 重要通知向けのアラートバリアントを追加する
  - ブランドカラー塗り・白文字の通知用バリアントを既存の成功・エラーバリアントとは別に追加する
  - 既存の成功・エラー表示の見た目・呼び出し方法を変更しない
  - 新バリアントを指定したアラートがブランドカラー地の帯として表示される状態を確認できる
  - _Requirements: 1.3, 2.4_
  - _Boundary: Alert_

- [x] 3. (P) Core: DAISOロゴ表示用コンポーネントを新規作成する
  - 山型のシンボルモチーフとワードマークを組み合わせた表示専用コンポーネントを新規に作成する
  - 実際の商標画像は使用せず、幾何学的な自作モチーフとして表現する
  - 色がテーマの配色設定に自動追従し、ブランドカラーで表示される状態になる
  - 単体でレンダリングした際にシンボルとワードマークの両方が表示される状態を確認できる
  - _Requirements: 3.1_
  - _Boundary: Logo_

- [x] 4. Integration: ヘッダー・サイドバー・言語切替へのブランド反映
- [x] 4.1 ヘッダーに新規ロゴコンポーネントを配置する
  - 既存のヘッダータイトル文言をロゴに付随する形で表示し直す
  - レイアウト・言語切替の配置を変更せず、ロゴ追加後もヘッダーの高さ・構造が崩れない状態になる
  - ヘッダーを開いた際にロゴとタイトル文言の両方が視認できる状態を確認できる
  - _Requirements: 1.3, 3.1_
  - _Depends: 3_

- [x] 4.2 (P) サイドバーのアクティブ項目表示をブランドカラー塗りに変更する
  - 現在選択中のナビゲーション項目をブランドカラー塗り・白文字で強調する
  - 非選択項目にマウスホバーした際は淡色のブランドカラー背景で反応する既存動作を維持する
  - サイドバーのいずれかの項目を選択した状態で、その項目だけが塗りで強調される状態を確認できる
  - _Requirements: 1.3, 3.2, 3.3_
  - _Boundary: Sidebar_

- [x] 4.3 (P) 言語切替のアクティブ言語表示をブランドカラーに変更する
  - 現在選択中の言語ラベルの強調色をブランドカラーに変更する
  - 言語切替の配置・切替機能自体は変更しない
  - 日本語・英語のいずれかに切り替えた際、選択中の言語ラベルがブランドカラーで強調表示される状態を確認できる
  - _Requirements: 1.3, 3.4_
  - _Boundary: LanguageSwitcher_

- [x] 5. Validation: 全画面のビジュアル整合性・アクセシビリティ確認
- [x] 5.1 ビルド・Lint・型チェックを実行し変更が壊れていないことを確認する
  - デザイントークン・コンポーネント変更後にビルド・Lint・型チェックを実行する
  - すべてエラーなく完了する状態を確認できる
  - _Requirements: 1.4_

- [x] 5.2 コントラスト比と既存バリアントの区別を確認する
  - 本文文字と背景、ブランドカラー地の白文字についてコントラスト比を確認する
  - エラー・成功・その他既存バッジ/アラートバリアントがブランドカラーと視覚的に混同されないことを確認する
  - コントラスト比がWCAG AA相当を満たしている状態を確認できる
  - _Requirements: 1.2, 5.1, 5.2_

- [x] 5.3 全機能ページで日英・レスポンシブ表示を確認する
  - ダッシュボード・問い合わせフォーム・問い合わせ一覧・お知らせ・リンク集・FAQの各画面を日本語・英語表示で確認する
  - FAQのカテゴリ別アコーディオン、リンク集のカテゴリ表示にブランドカラーが一貫して反映されていることを確認する
  - カード表示の見た目（薄枠・最小限の影）が変更前と同じままであることを確認する
  - 翻訳文言・翻訳キーに変更が入っていないことを確認する
  - タブレット幅・PC幅で既存のレスポンシブ挙動が崩れていない状態を確認できる
