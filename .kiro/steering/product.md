# プロダクト概要

## プロジェクト概要

- **対象**: 日本の大手小売店内プロジェクト。利用者は海外の販社（20か国以上）担当者
- **課題**: 現状メールベースの問い合わせ・申請対応が非構造化で、日本のヘルプデスク側の分類作業が負担になっている
- **目的**: 問い合わせ・申請を統一フォーム化し、選択式項目で自動分類。加えて情報共有・リンク集を1つのポータルに集約する

## 開発フェーズ方針

| フェーズ | 概要 |
|---|---|
| フェーズ1（現在） | Next.jsによるフロントエンドのみのモックアップ作成。バックエンド・DBは未実装 |
| フェーズ2（今後） | ヘルプデスク担当者へのヒアリング結果を反映し、モックアップを修正 |
| フェーズ3（今後） | DB・バックエンドAPIを実装し、モックAPIを実APIに差し替え |

## 主要機能

| 機能 | 概要 |
|---|---|
| ダッシュボード | お知らせ・問い合わせステータス概要 |
| 問い合わせ・申請フォーム | 選択式項目＋自由記述。分類・緊急度・地域などを選択式で入力し分類作業を削減 |
| 問い合わせ一覧 | 自分（自社）が出した問い合わせの状況確認 |
| お知らせ・情報共有 | ヘルプデスクからの周知事項 |
| リンク集 | カテゴリ別の外部/内部サイトリンク |
| FAQ | よくある質問 |

## 機能実装推奨順序

1. `dashboard`（土台となるレイアウト・ナビゲーション）
2. `inquiry-form`（最重要機能。フォーム項目・分類軸を型に落とし込む）
3. `announcements`（お知らせ・情報共有）
4. `links-page`（リンク集）
5. `faq`（FAQ）
